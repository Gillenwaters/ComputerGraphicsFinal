Relevant Functions

The following is a brief explanation of functions used in our sample implementation, listed in order of appearance. This summary is not exhaustive and may not include all available parameters.

This information was primarily derived from the open-source Mozilla Developer Network (MDN) web docs for WebGLRenderingContext, with some information included from MDN's Document and HTMLCanvasElement documentation pages. For more information and details, visit these sources.

document.querySelector(selectors) - a Document method that accepts as input a DOMString consisting of a valid CSS selector string. returns the first Element object within the document matching the specified selector or group of selectors. If no matches are found, null is returned

canvas.getContext(contextType) - an HTMLCanvasElement method that accepts as input a DOMString consisting of the context identifier defining the drawing context associated to the canvas - in our case we are specifying “webgl”. returns either a drawing context on the canvas. will return null if context identifier is unsupported or canvas has already been set to a different context mode

gl.createShader(type) - a WebGLRenderingContext method accepts input of type gl.VERTEX_SHADER or gl.FRAGMENT_SHADER. returns a modifiable WebGL shader

gl.getShaderParameter(shader, pname) - a WebGLRenderingContext method accepts input specifying the shader and information required. when COMPILE_STATUS is entered as parameter (pname), returns a GLboolean indicating whether or not the last shader compilation was successful

gl.createProgram() - a WebGLRenderingContext method that creates and initializes a WebGLProgram object, linking compiled GLSL vertex shader and GLSL fragment shader into a usable program

gl.AttachShader(program, shader) - a WebGLRenderingContext method that specifies shaders to be included in link operations

gl.LinkProgram(program) - a WebGLRenderingContext method that links the specified program given the vertex shader and fragment shader are declared appropriately and requirements are met

gl.getProgramParameter(program, pname) - a WebGLRenderingContext method accepts input specifying the program and information required. when LINK_STATUS is entered as parameter (pname), returns a GLboolean indicating whether or not the last link operation was successful

gl.createBuffer() - a WebGLRenderingContext method that stores data which might include vertices or colors

gl.bindBuffer(target, buffer) - a WebGLRenderingContext method that specifies a target binding point, such as an ARRAY_BUFFER containing vertex indices, or an ELEMENT_ARRAY_BUFFER containing the indices of elements, that should be bound to a WebGL buffer and will serve to define the behavior of the associated objects

Math.floor() - a built-in Javascript function that returns the largest integer less than or equal to a given number

gl.bufferData(target, size, usage) - a WebGLRenderingContext method that initializes and creates the buffer object's data store where the target binding point may include an ARRAY_BUFFER or ELEMENT_ARRAY_BUFFER, the size is specified in bytes, and usage may include STATIC_DRAW, STREAM_DRAW, or DYNAMIC_DRAW
